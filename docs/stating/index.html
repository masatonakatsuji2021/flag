- Operation flow

Flag will carry out front-end application development with the following flow.

  • 1. Create a Project
  • 2. Writing and modifying code
  • 3. Run a build of the project
  • 4. Browsing confirmation of build data
  • 5. (For Android and desktop apps) Build or debug a terminal application


Up to the deployment to the web server,
the flow is from 1 to 4, and in the case of Android/Window application development, 5 is included.

- Create a Project

A project in Flag refers to a unit of application development.
Vendors must always start by creating individual projects first.

Projects are basically created by command execution.
Of course, it is theoretically possible to create from scratch without command execution,
but you will have to prepare the necessary files manually.
It is quite troublesome and time consuming, so it is recommended to create a project by command because it is the easiest.

To create a project, execute the following command on any directory after installing Flag.

flag front init

Starting the command is simple, and after execution, various input prompts are displayed interactively as shown below.
The input prompt asks about the name of the project, whether it supports TypeScript description, and the framework to use (Cordova, etc.)
See here for more information on prompts and more.

- Build the Project

The source in the project consists of multiple HTML files and js files (or ts files if using TypeScript).
A build process is required to make it viewable on a web browser.

The build merges the script files into a single js file.
Convert multiple HTML files (views or templates) into script code at the same time.
This allows for SPA (Single-Page-Action) that doesn't require an actual page transition.

Build data is output to the frameworks/web directory within the project.
This directory will be the default build output destination.This directory will be the default build output destination.
Apart from this, if you add various frameworks, the build data will be duplicated in each directory.

To build, just run the following command in the project directory.

flag front Build

After that, the merging and creation of each file is done automatically.

If you enable TypeScript, the ts file will be built at the same time.
What is actually merged is the js file transcompiled from the ts file.
For TypeScript, see here.

- Support for other frameworks (Cordova, Electron)

The default build of Flag only supports browser viewing.
Apart from this, it is possible to link with application development frameworks such as Cordova and Electron.

For the supported frameworks (Cordova, Electron), a project area for the framework is created in the frameworks directory at the same time as the build data.
When executing a build, a set of HTML source files in the added framework is also duplicated at the same time, so
This makes it possible to smoothly develop applications for each framework.
